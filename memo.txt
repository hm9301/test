git 버전 확인
git --version
프로젝트 디렉토리에 로컬 저장소 생성
git init
git 버전 관리할 파일을 선택: stage에 올림
git add test.py
버전 만들기
git commit -m "test.py를 커밋"
로그 확인
git log
로컬 저장소에서 github 원격 저장소 주소를 등록
git remote add origin https://github.com/hm9301/test.git
생성한 커밋들을 github에 전송
git push -u origin master
Fork
하나의 저장소를 다른 계정으로 복사
git 명령어가 아님(github 기능)
> 다른 사람의 github 계정 저장소에서 fork를 눌러 내 계정 저장소로 복사
README.md
마크다운 형식으로 작성
> push하면 github에 메인에 반영
원격 저장소에서 변경된 사항을 가져오기
git fetch origin (가져오기만)
git pull origin(가져와서 merge)
.gitignore
git이 추적하지 않아야 할 파일을 지정하는 파일
예시) 파일이름.확장명
*.확장명
폴더명/
branch
작업에 대한 가지
큰 줄기에 영향을 주지 않고 가지에서만 작업을 할 수 있음
각 가지는 자체 커밋 및 변경을 가질 수 있음
> master에 영향을 미치지 않고 여러 기능이나 버그 수정을 동시에 수행할 수 있음
git status
git branch
apple을 생성하고 이동하기
git checkout -b apple
